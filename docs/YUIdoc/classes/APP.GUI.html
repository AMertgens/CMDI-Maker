Creates a language search form, where the user can search a language or type in an ISO639-3 code to add a language somewhere.

Parameters:

    Parent element in DOM

  • element_id_prefix String

    Prefix of the IDs of all elements that are created here.

    Callback function that is called, when the user adds a language.

openVocabulary

(
  • parent
  • title
  • name
  • id
  • options
  • value
  • hover
  • className
)

Defined in core\app.gui.js:252

Creates a user input field where the user either can select an option from a given vocabulary or type in a custom option. As always, there is a heading above the element.

Parameters:

  • parent Object

    Parent element in DOM

  • title String

    Title or heading that is displayed above the form element

  • name String

    Name of the input elements

  • id String

    ID of the input elements. Note that both select and input will have the same ID.

  • options Object

    Array with strings. Vocabulary of values.

  • value String

    Value of the input field. Note that if the value is part of options, the select element will be used, otherwise the text input element.

  • hover String

    HTML Title Element value. This value is displayed as a tooltip when mouse is hovering above the element.

  • className String

    Class name of the select and input elements.

Returns:

return_object Returns an object that contains both the select and the input element. Keys are "text" and "select".
